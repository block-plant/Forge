Guide & FAQ

            
Everything you need to know about connecting Forge to your project.

          


          

            
What is Forge?

            
Forge is a self-hosted Backend-as-a-Service (BaaS) — think of it as your own private Firebase. It gives your app a complete backend with zero cloud lock-in. You host it, you own it, your data stays with you.

          


          
🧩 What Forge Adds to Your Project

          

            

              
🔐

              
Authentication

              
Ready-to-use user signup, login, and session management. Supports email/password auth with JWT tokens — no need to build auth from scratch.

            

            

              
🗄️

              
Database

              
A document-based NoSQL database with collections and documents — similar to Firestore. Store, query, and organize your app's data with simple REST calls.

            

            

              
📁

              
File Storage

              
Upload and serve files (images, PDFs, videos) via a simple API. Each file gets a permanent URL you can use directly in your app.

            

            

              
📊

              
Analytics

              
Track custom events from your app (page views, button clicks, purchases). See real-time counters and daily stats — no third-party scripts needed.

            

            

              
⚡

              
Real-Time Updates

              
WebSocket-powered live data sync. Subscribe to database changes and get instant updates pushed to your app — perfect for chat, dashboards, and collaboration tools.

            

            

              
🌐

              
Static Hosting

              
Deploy your frontend (HTML, CSS, JS) directly on Forge. It serves your site with proper MIME types, caching, and gzip — no separate hosting needed.

            

          


          

            
🚀 Quick Start — Connect in 60 Seconds

            

              
Step 1: Find your Forge endpoint URL (shown in Settings, e.g., http://your-server:8080).

              
Step 2: Create a user account:

              
curl -X POST http://your-server:8080/auth/signup \
  -H "Content-Type: application/json" \
  -d '{"email":"you@example.com","password":"your-password"}'

              
Step 3: Start using the database:

              
curl -X POST http://your-server:8080/db/todos \
  -H "Content-Type: application/json" \
  -d '{"task":"Build something great","done":false}'

              
Step 4: Read it back:

              
curl http://your-server:8080/db/todos

            

          


          

            
❓ Frequently Asked Questions

            

              

                
How is Forge different from Firebase? ▼

                
Firebase is a cloud service owned by Google — your data lives on their servers. Forge is self-hosted, meaning you run it on your own machine or VPS. You get the same features (auth, database, storage, analytics) but with full ownership of your data and zero recurring costs beyond your server.

              

              

                
What languages/frameworks can I use with Forge? ▼

                
Any language that can make HTTP requests. Forge exposes a REST API, so it works with JavaScript, Python, Go, Swift, Kotlin, Flutter, React Native, or any other language. Just make fetch() or requests.post() calls to your Forge server.

              

              

                
Do I need to install anything on my app? ▼

                
No SDK required. Forge uses standard REST APIs. You can use plain fetch() in JavaScript, the requests library in Python, or curl from the terminal. We plan to release optional SDKs for convenience, but they're not required.

              

              

                
How do I secure my data? ▼

                
Forge includes built-in authentication with JWT tokens. Users sign up and log in through the auth endpoints. You can protect your API by passing the JWT token in the Authorization header. Rate limiting and CORS are also built in.

              

              

                
Can I run multiple projects on one server? ▼

                
Yes! The Admin Console (port 8080) can deploy and manage multiple isolated project instances, each running on its own port (8081–8100). Each project gets its own database, storage, and auth — completely independent from each other.

              

              

                
What are the system requirements? ▼

                
Forge is written in pure Go with zero external dependencies. It runs on any Linux server, Mac, or Windows machine. A small VPS (1 CPU, 1 GB RAM) is enough for development and light production use.

              

              

                
Where does my data get stored? ▼

                
All data is stored locally on your server's filesystem. Database documents, uploaded files, and user accounts are saved in the project's data directory (typically /var/lib/forge-data/). Nothing ever leaves your server.
